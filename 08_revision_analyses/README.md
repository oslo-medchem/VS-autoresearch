# 08_revision_analyses

Deterministic re-analyses added during the **Molecular Informatics** major-revision
round (no new docking). Each script maps to a reviewer point and operates on data
already generated by the campaigns/pilots (see `../` stage directories and the
project data archive). Scripts take the archive path as the first argument and
write to an `analysis_outputs/` directory.

| Script | Reviewer point | What it does |
|---|---|---|
| `library_descriptors.py` | R1.4 (library description) | RDKit physicochemical descriptor panel (MW, logP, HBD/HBA, TPSA, rotatable bonds, aromatic rings, QED, Fsp3, Lipinski violations) for the CDK2 and pilot libraries, summarised by active/decoy label. |
| `fpr2_actives_descriptors.py` | R1.4 (library description) | Regenerates the FPR2 **active** set directly from its defining ChEMBL query (target CHEMBL4227, pChEMBL >= 5, EC50/IC50/Ki/Kd) and computes the same descriptor panel, because the archived FPR2 library SMILES files are null-filled. Characterises the full ChEMBL active chemical space (n = 2,118), not the 573-compound docked subset. Output in `analysis_outputs/`. |
| `cdk2_filter_lossbias.py` | R3.3, R3.4 | Active-vs-decoy loss analysis of the CDK2 PAINS/Brenk (skill) filter: recovers which compounds survived each preparation and Fisher-tests whether actives were dropped at a different rate than decoys. |
| `pilot_thirdtarget_stats.py` | R1.3, R3.7 | Third-target (HIV-1 protease) Vinardo-minus-Vina deltas from the pilot data, as an independent check of the target- and metric-dependent scoring-function behaviour. |
| `redock_cocrystal.py` | R1.4 (pose validation) | Gated co-crystal self-redock (AutoDock Vina) computing heavy-atom RMSD of the top pose to the experimental ligand. Run for FPR2 (FUI into 7T6S): top pose 0.37 Å RMSD. |

## Key corrected result (R3.3)

`cdk2_filter_lossbias.py` established that the CDK2 skill (PAINS/Brenk) filter dropped
**37.8% of actives (179/474) versus 49.3% of decoys (984/1,997)**, so it *enriched* the
retained set in actives (Fisher exact odds ratio 0.63 for an active being dropped,
p = 6.5 × 10⁻⁶). The higher AUC of the unfiltered (naive) library therefore reflects a
change in benchmark composition, not a genuine docking-protocol improvement. This
corrects the original draft's statement that the filter "biased the benchmark against the
filtered library"; see `../REVISION_NOTES.md`.
